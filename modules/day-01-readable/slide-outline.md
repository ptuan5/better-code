# Slide Outline

## Slide 1: Why Readability Matters

- code is read more often than it is written
- readability supports debugging, review, and collaboration

## Slide 2: Common Signals of Unreadable Code

- vague names
- mixed responsibilities
- long blocks with no structure
- comments that do not explain intent

## Slide 3: Project Organization

- separate data, code, and outputs
- use stable, descriptive folder names
- make navigation obvious to a newcomer

## Slide 4: Better Naming and Comments

- choose names that communicate role and units
- comment on why, not just what
- use docstrings or headers for script purpose

## Slide 5: Activity Setup

- activity 1: reorganize a messy project so a newcomer can navigate it
- inspect the starter script
- improve clarity without changing behavior
- be ready to explain your choices
