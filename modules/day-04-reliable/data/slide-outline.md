# Slide Outline

## Slide 1: What Reliability Means

- correct results
- predictable behavior
- visible assumptions

## Slide 2: Review Mindset

- inspect logic before running code
- ask what could fail silently
- look for fragile edge cases

## Slide 3: What to Test First

- core calculations
- branch logic
- empty or unusual inputs

## Slide 4: Small Tests, Big Payoff

- tests document expectations
- tests reduce fear of change
- tests catch regressions early

## Slide 5: Activity Setup

- review first
- fix the code
- add a few meaningful tests
