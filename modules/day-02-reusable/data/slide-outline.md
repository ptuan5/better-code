# Slide Outline

## Slide 1: Why Reuse Matters

- repeated code is harder to maintain
- one fix should not require five edits

## Slide 2: Code Smells That Suggest Refactoring

- copy-pasted blocks
- repeated literals
- nearly identical filtering steps
- repeated plotting or reporting logic

## Slide 3: Good Function Design

- one clear responsibility
- descriptive name
- useful parameters
- sensible return value

## Slide 4: Parameterization Without Overengineering

- make real variation explicit
- keep defaults simple
- resist adding arguments you do not need yet

## Slide 5: Activity Setup

- identify repeated logic
- refactor to functions
- explain your design choices
