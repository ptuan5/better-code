# Activity 2: Code Peer Review

## Goal

Review a partner's improved materials and focus on whether a newcomer could actually use them.

## Scenario

Your partner has cleaned up their starter repo, and now you are acting as the first real outside user. You are not reviewing style for style's sake. You are checking whether a newcomer could figure out the purpose, find the run steps, and trust the handoff.

Good review here means noticing both what already works and what would still block a real user.

This review is the final newcomer test for the week: can someone understand it, reuse it, rerun it, and trust it well enough that sharing actually works?

## Files

- your partner's revised starter project from `../activity-1/`
- checklist below for review

## Task

Review a partner's improved version and focus on whether a newcomer could use it.

1. Check whether the purpose of the script is obvious.
2. Check whether the inputs, outputs, and run steps are easy to find across the whole folder.
3. Leave one note about what is already clear.
4. Leave one note about what would still block a new user.
5. If time allows, suggest one concrete README sentence or heading that would improve the handoff.

## Peer Review Checklist

- Is there a clear script purpose?
- Is there a clear file a newcomer should open first?
- Are the main inputs and outputs named well?
- Would you know where results are written?
- Could you rerun it without asking the author a question first?
- Does the documentation match the actual script behavior?
