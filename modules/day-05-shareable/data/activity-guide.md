# Activity Guide

Theme: Make your work usable by others.

## First 75 Minutes

### 1. Why Shareability Matters

Time: 15 minutes

- discuss what makes code approachable to a new user
- connect sharing to the previous four course themes

### 2. Concept and Demo: GitHub and Basic Repo Structure

Time: 15 minutes

- review the role of a README, clear structure, and visible run instructions
- show a stronger and weaker example side by side

### 3. Activity 1: Fix This Repo and Prepare It for GitHub

Time: 25 minutes

1. Identify the most confusing parts of the starter materials.
2. Rewrite headers, file names, or README-style guidance.
3. Make the code easier for another researcher to run or adapt.

### 4. Pull Request Concept and Assignment

Time: 20 minutes

- explain what a pull request is for
- decide what kind of feedback you would request
- prepare to exchange work with a partner

## Break

Time: 15 minutes

## Final 60 Minutes

### 5. Activity 2: Peer Review

Time: 20 minutes

1. Review a partner's improved materials.
2. Leave clear, specific feedback.
3. Focus on what would help a newcomer most.

### 6. Showcase and Reflection

Time: 30 minutes

- invite 2-3 volunteers to present one improvement
- reflect on what changed most across the week

### 7. Final Wrap-Up

Time: 10 minutes

- summarize the link between readable, reusable, reproducible, reliable, and shareable code
- identify one practice participants will use next week
