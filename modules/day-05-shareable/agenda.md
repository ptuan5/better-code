# Agenda

Theme: Make your work usable by others.

### 1. Why Shareable & What It Entails

Time: 15 minutes

- discuss what makes code approachable to a new user
- connect sharing to the previous four course themes

### 2. Demo: GitHub & Basic Repo Structure

Time: 15 minutes

- review the role of a README, clear structure, and visible run instructions
- show a stronger and weaker example side by side

### 3. Activity 1: Fix This Repo & Publish to GitHub

Time: 25 minutes

1. Open `activities/activity-1/prompt.md`.
2. Identify the most confusing parts of `activities/activity-1/r/starter.R` or `activities/activity-1/python/starter.py`.
3. Rewrite headers, file names, or README-style guidance.
4. Make the code easier for another researcher to run or adapt.

### 4. Pull Request Concept and Assignment

Time: 20 minutes

- explain what a pull request is for
- decide what kind of feedback you would request
- prepare to exchange work with a partner

## Break

Time: 15 minutes

### 5. Activity 2: Code Peer Review

Time: 20 minutes

1. Open `activities/activity-2/prompt.md`.
2. Use the checklist to review a partner's improved materials.
3. Leave clear, specific feedback.
4. Focus on what would help a newcomer most.

### 6. Showcase + Reflection

Time: 30 minutes

- invite 2-3 volunteers to present one improvement
- reflect on what changed most across the week

### 7. Workshop Wrap Up / Buffer Time

Time: 10 minutes

- summarize the link between readable, reusable, reproducible, reliable, and shareable code
- identify one practice participants will use next week
