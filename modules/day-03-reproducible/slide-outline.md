# Slide Outline

## Slide 1: What Breaks Reproducibility

- undocumented dependencies
- machine-specific paths
- unclear run order
- manual steps that no one records

## Slide 2: Minimum Reproducible Habits

- record packages and versions
- make inputs and outputs visible
- separate setup from analysis
- provide run instructions

## Slide 3: From Script to Workflow

- define a clear entry point
- use small named steps
- produce predictable outputs

## Slide 4: Environment and Execution

- lock or document dependencies
- keep file paths portable
- decide what should be automated

## Slide 5: Activity Setup

- inspect hidden assumptions
- rewrite for repeatability
- document how to rerun the work
