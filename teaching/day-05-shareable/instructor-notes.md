# Instructor Notes

## Session Goal

Help participants package their work so another researcher can understand and reuse it quickly.

## Materials to Open

- `modules/day-05-shareable/agenda.md`
- `modules/day-05-shareable/activities/activity-1/prompt.md`
- `modules/day-05-shareable/activities/activity-2/prompt.md`
- `modules/day-05-shareable/activities/activity-1/r/starter.R` or `modules/day-05-shareable/activities/activity-1/python/starter.py`
- `modules/day-05-shareable/activities/activity-1/r/example.R` or `modules/day-05-shareable/activities/activity-1/python/example.py`

## Key Talking Points

- sharing begins with communication, not just version control
- a strong README answers what, why, and how
- GitHub makes collaboration easier when the workflow is explicit
- polishing a project means reducing friction for the next person

## Prompts

- What would you need to know before reusing someone else's code?
- Which README sections are most important for this audience?
- How does pull request review support better collaboration?

## Facilitation Tips

- Use real examples of weak and strong README language.
- Encourage learners to write for a specific audience rather than everyone.
- Keep GitHub workflow explanations concrete and lightweight.

## Debrief Questions

- Which README improvement would help a newcomer the most?
- What project detail is most often forgotten during sharing?
- How did this module connect to the previous four days?
