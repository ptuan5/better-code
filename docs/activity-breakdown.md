# Activity Breakdown

Source: `Activity Time Breakdown.docx`

## Day 1: Readable

Theme: Write code you will understand six months later.

### Suggested Flow

1. 15 min: intro, course objectives, and structure
2. 20 min: why readable code matters and how project structure helps
3. 15 min: activity to organize a directory
4. 15 min: discussion
5. 15 min: break
6. 15 min: code-level readability, naming, docstrings, linting
7. 20 min: activity to rewrite confusing code
8. 20 min: discussion
9. 5 min: wrap-up and Q&A

## Day 2: Reusable

Theme: Don't repeat yourself.

### Module 2A

1. 15 min: why DRY matters and common code smells
2. 15 min: demo of refactoring repeated logic into functions
3. 30 min: activity to identify repetition and extract 1-2 functions
4. 15 min: discussion
5. 15 min: break

### Module 2B

1. 10-15 min: notebook parameterization concept
2. 30 min: activity to fix a messy notebook
3. 15-20 min: discussion

## Day 3: Reproducibility

Theme: Someone else, or future you, can rerun everything.

### Module 3A: Environments

1. 15 min: why code breaks and why environments matter
2. 20 min: demo of an `environment.yml`
3. 30 min: activity to reconstruct an environment
4. 10 min: discussion

### Module 3B: Scripts to Pipelines

1. 15 min: problems with ad hoc scripts
2. 25 min: activity to define execution order and modularize one script
3. 15-20 min: discussion

## Day 4: Reliability

Theme: Your code is wrong; design for that.

### Module 4A: Code Review Mindset

1. 15 min: what to review
2. 20 min: demo of subtle bugs and fragile logic
3. 30 min: peer review activity using a checklist
4. 10 min: discussion

### Module 4B: Testing

1. 15 min: why testing matters
2. 25-30 min: write first tests for a simple function
3. 15-20 min: discussion

## Day 5: Shareable Code

Theme: Make your work usable by others.

### Module 5A: GitHub and Collaboration

1. 15 min: why sharing matters
2. 20 min: demo of good vs bad READMEs and repo structure
3. 30 min: activity to improve a messy repo
4. 10 min: discussion

### Module 5B: Showcase and Reflection

1. 40 min: polish one earlier script or notebook
2. 20 min: showcase and reflection
