# Activity Breakdown

Source: `Activity Time Breakdown.docx`

## Day 1: Readable

- intro, objectives, and structure
- discuss why readable code matters
- organize a messy directory
- rewrite confusing code
- discuss cleanup choices

## Day 2: Reusable

- discuss DRY and code smells
- refactor repeated logic into functions
- clean up a messy notebook by parameterizing key values
- discuss what should stay flexible and what should stay simple

## Day 3: Reproducible

- discuss why code breaks across machines
- review environment files
- reconstruct dependencies
- define execution order across scripts

## Day 4: Reliable

- discuss code review mindset
- inspect buggy or fragile code
- do peer review on cleaned-up code
- write first tests for a small function

## Day 5: Shareable

- discuss what makes a repository usable
- compare stronger and weaker README examples
- improve a messy repo
- polish one earlier script or notebook for showcase
