# Day 4: Reliable

## Theme

Assume your initial code is wrong: always revise.

## Learning Focus

- reviewing correctness, assumptions, and edge cases
- using peer review to improve code quality
- writing simple tests with `testthat` or `pytest`

## Modules

### 4A: Code Review

- concept: what reviewers should look for
- demo: inspect code with subtle bugs
- activity: write review comments using a checklist

### 4B: Testing

- concept: normal, edge, and failure cases
- activity: write 2-3 tests for a small function
- discussion: what is worth testing
