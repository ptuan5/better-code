# Day 2: Reusable

## Theme

Don't repeat yourself: functions are your friend.

## Learning Focus

- spotting repetition and code smells
- designing functions around clear inputs and outputs
- keeping function scope small
- parameterizing notebooks

## Modules

### 2A: Refactoring Repetition

- demo: repeated logic turned into functions
- activity: identify repetition and extract 1-2 functions
- discussion: what should and should not become a function

### 2B: Notebook Templates

- concept: why notebooks become fragile
- activity: move parameters to the top and make a notebook reusable
- discussion: what still feels brittle
